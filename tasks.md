開発タスクリスト（実装順）
- Canvasレース単ページの雛形作成（Next.js page + 基本レイアウト/Canvas設置）
- フロント内レースロジック実装（距離/速度更新、接触補正、ゴール判定）
- Canvas描画実装（楕円トラック、プレイヤー位置計算と描画、requestAnimationFrameループ）
- UIでのプレイヤー2名ステータス入力とレース開始フロー連結
- 勝者表示と簡易リセット/再走機能
- （余裕）FastAPIエンドポイントの枠だけ用意（/players, /races/result）とSQLite接続セットアップ

Definition of Done
- 雛形: Next.jsで単ページ表示、Canvas要素が表示される
- ロジック: 速度/距離が毎フレーム更新され、ゴール判定が機能する
- 描画: トラック楕円と2つの丸が動き、周回が可視化される
- 入力/開始: ステータス入力→「レース開始」でロジックがその値を使用し進行
- 勝者表示: ゴール後に勝者名/色を表示し、リセットで再度開始できる
- FastAPI枠: サーバー起動し、ダミーで /players (GET/POST) と /races/result (POST) が200を返す; SQLite接続が初期化される

MVPに含めない後回し
- データ永続化/過去レース一覧
- トーナメントや複数レース管理
- 複雑な物理モデルやチューニング
- マルチプレイ/同期
- プレイヤー大量生成やシード再現

最小構成案
- frontend: Next.js単ページ, Canvas描画, React stateにプレイヤー2体のステータス, クライアント内レースシミュレーション, requestAnimationFrameループ
- backend: FastAPI + SQLite（必要ならplayers/racesテーブル）; CORS許可; MVPはフロント単独で動作、APIは薄い枠のみ

まず作るべき最初の画面と理由
- プレイヤー2体のステータス入力＋Canvas表示＋「レース開始」ボタンを1ページで置いた画面。入力→動作→勝者表示までの最短導線を一度に検証でき、バックエンド依存を避けられる。
